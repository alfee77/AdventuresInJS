Methods of primitives

Quick reminder as to what the primitive data types in JavaScript are:

string, number, bigint, boolean, symbol, null and undefined

JavaScript allows us to work with primitives as if they were objects - epic :). However, primitives are not objects. Primitive variables are capable of storing a single data type, unlike objects which can have multiple values as properties. One of the best things about an object is we can store a function as a property:

let myCar = { manufacturer: "Kia", model: "Niro", Date of registration: 30/09/2021, tootHorn(){ alert(`Beep beep! said the ${this.manufacturer} ${this.model}`); } } function example1(){ myCar.tootHorn(); }

So here's an example of an object, myCar{...} with the method tootHorn(). I mean what kind of a car doesn't have a horn eh? One must be able to toot.

Many inbuilt objects exist in JavaScript. For example, Dates, errors, DOM and HTML elements etc. Much like myCar, these objects come with their own properties, many of which are methods. However, it must be born in mind that objects are "heavier" than primitives. They require a bit more "under the hood" to come into existence.

So the paradox facing the original developer of JavaScript is; yeh - lets have primitive data types that hold simple/low overhead variables, but how do we give these things methods to carry day to day operations on it?

The solution looks a little bit awkward, but here it is:

  • Primitives are still primitive. A single value, as desired.
  • The language allows access to methods and properties of strings, numbers, booleans and symbols.
  • In order for that to work, a special “object wrapper” that provides the extra functionality is created, and then is destroyed.

To allow access to the methods and properties of these primitive data types, the language provides wrapper objects, namely String, Number, Bigint, Boolean and Symbol. Consider the following which implicitly cracks into the String object wrapper:

let str = "Hello"; alert( str.toUpperCase() ); // HELLO

Note it is not necessary to explicitly wrap the String in the object wrapper; the methods are available implicitly.

Tasks

Task 1 - adding properties to a primitive?

Consider the following code:

function task1(){ let hello = "hi"; hello.test = 5; alert(hello.test); }

What do you think? Will it work?

Solution

If the 'use strict' directive is used, running this code will result in an error. Otherwise, running this code will post "undefined" in the alert box. This is because the object wrapper is created when the value 5 is added to the hello.test property, however as soon as the value is applied the object wrapper is disposed of.
