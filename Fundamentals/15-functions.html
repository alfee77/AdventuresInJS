Functions

To create a function, it needs to be declared. It would typically look like this:


    function message(){
        alert(`Hello everyone!`);
    }

The function keyword goes first, followed by the function name, followed by a list of comma separated parameters between the parenthesis. The parameters are essentially local variables that are passed to the function when they are formed. The parameters can also have default values. The function body then follows between curly braces {}.

A word on local and global variables

A local variable is declared within a function, and a global function is declared outwith the function. This then leads to a question of scope; the local variable can only be "reached" from within the function. A global variable can be accessed from anywhere in the body of the code.

Consider the following code:


    function outerFruit(){
        let fruit = "banana";
                    
        eatFruit();

            let vegetable = `tomato`;
            alert(`I like ${fruit}!`);
            alert(`Is ${vegetable} really a vegatable? Or is it a fruit?`);
        }
        try{
            alert(`Not so sure about ${vegetable}`);
        }catch(err){
            alert(`Just not possible! A ${err.name} has been thrown. ${err.message}`);
        }
    }

When the last line of code is run, a ReferenceError is thrown, confirming that vegetable is not defined.

Parameters

Parameters are passed to functions when they are called; they are the variables stated in the brackets of the function. Consider the following piece of code:


    function announce(title, name){
        alert(`Welcome ${title} ${name}`);
    }

In summary

Functions may have local variables: those declared inside its body or its parameter list. Such variables are only visible inside the function. Parameters can have default values: function sum(a = 1, b = 2) {...} . Functions always return something. If there’s no return statement, then the result is undefined.
