Function Expressions

The classical way to write a function looks like this:


    function sayHi(){ 
        alert(`Hi!`); 
    }

The function is declared, assigned a name, any arguments are defined in the parenthesis, and the body of the function's code is embedded in the curly brackets {}. This code may then be assigned to the onclick attribute of an HTML button, and off you go!

Click this button to say "Hi!"

However, it is also possible to create a function in runtime "on the hoof". Examine the following code:


    let sayHello = function(){ 
        alert(`Hello!`);
    }

This function is created on the assignment side of an expression (i.e. to the right hand side of the =), and is immediately assigned to the variable sayHello.

At first glance it may appear that this is two different ways of achieving the same thing. Callback Functions

We can make use of this phenomena. When clicked the following button will ask the user for an answer, and depending on the answer given will direct the code to a given function.

The code looks like this:


    function ask(question, yes, no) { 
        let answer = confirm(question); 
        if (answer) yes() 
        else no();    
    } 
                      
    function clickedOk() { 
        alert( "You agreed." ); 
    } 
                      
    function clickedCancel() { 
        alert( "You canceled the execution." ); 
    }

When the function ask() is called, it is done so in the following form:


    ask('Do you agree?', clickedOK, clickedCancel);

Observations:

When the function ask() is declared, it is defined with 3 arguments; "question", "yes" and "no". Due to JavaScript's loosely typed nature, these arguments aren't typed when they are declared. The content of the function is very simple; present the user with a Yes/No "question", the text of which is the first argument passed to the ask function, and act depending on the answer. The answer is gathered by means of a confirmation box, delivered by the confirm() function. If the user clicks "Yes", the confirm() function returns truthy; in this condition the function yes() is called (the second argument passed to the function). If the user clicks "No", the confirm function returns falsy; in this condition the no() function is called (the third argument passed to the function).

This function call is added to the onclick attribute of the following button.
